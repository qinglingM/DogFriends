# Agent Checklist 索引

本文件用于告诉 AI / Agent：遇到不同任务时，应该读取哪份专项检查清单。

核心原则：  
不要全量读取所有规则。做什么任务，就读取什么 checklist。

## 一、UI / 交互类任务

当任务涉及以下内容时，读取：

- `checklists/UI_CHECKLIST.md`

触发场景：

- 页面布局
- 安全区
- 顶部栏 / 底部栏
- 输入框
- 评论框
- 搜索框
- 键盘遮挡
- 头像展示
- 头像裁剪
- 间距调整
- 字段展示
- 空状态
- 加载状态
- 成功 / 失败提示
- 防重复点击
- 列表滚动
- Tab 切换

## 二、数据 / 状态 / 删除类任务

当任务涉及以下内容时，读取：

- `checklists/DATA_CHECKLIST.md`

触发场景：

- 数据归属
- userId / profileId / ownerId / authorId
- 发布
- 删除
- 点赞
- 收藏
- 评论
- 刷新
- 下拉刷新
- 本地状态
- 数据库同步
- 硬编码数据
- 数据不一致
- 软删除 / 硬删除
- 列表数量不准确

## 三、账号体系类任务

当任务涉及以下内容时，读取：

- `checklists/AUTH_CHECKLIST.md`

触发场景：

- 注册
- 登录
- 退出登录
- 忘记密码
- 修改密码
- 注销账号
- 游客模式
- 手机号登录
- 验证码登录
- 第三方登录
- Apple 登录
- 微信登录
- 用户协议
- 隐私政策

## 四、图片 / 媒体类任务

当任务涉及以下内容时，读取：

- `checklists/MEDIA_CHECKLIST.md`

触发场景：

- 图片上传
- 头像上传
- 图片裁剪
- 图片压缩
- 图片格式
- 图片尺寸
- 图片存储
- 对象存储
- 图片 URL
- 上传失败
- 图片刷新
- 旧图缓存
- 默认图

如果任务同时涉及头像展示，也要读取：

- `checklists/UI_CHECKLIST.md`

## 五、发布 / 构建 / 环境类任务

当任务涉及以下内容时，读取：

- `checklists/RELEASE_CHECKLIST.md`

触发场景：

- Git
- commit
- push
- 分支
- 构建
- 打包
- 部署
- 环境变量
- 服务器
- 数据库迁移
- 正式环境
- 测试环境
- 回滚
- App 上架
- 白屏
- 构建失败

## 六、完整规则库

当出现以下情况时，才读取：

- `AGENT_RULES_FULL.md`

触发场景：

- 规则冲突
- 需要重新整理规范
- 需要确认某条规则来源
- 用户要求查看完整协作规范
- 需要补充新规则
- Agent 对短规则理解不清

## 七、禁止行为

除非用户明确要求，否则不要：

1. 每次任务都读取完整规则库。
2. 每次任务都读取所有 checklist。
3. 把沟通规则和专项检查项混在一起输出。
4. 为了“显得全面”输出大量无关检查项。
5. 在缺少关键信息时直接替用户做决定。
