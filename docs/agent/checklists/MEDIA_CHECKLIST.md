# Media Checklist

本文件用于处理图片上传、头像上传、图片裁剪、压缩、格式、存储和刷新问题。

## 一、图片上传前检查

涉及图片上传时，必须明确：

- 最大上传大小
- 支持格式
- 是否压缩
- 压缩后尺寸
- 压缩后质量
- 是否保留原图
- 是否生成缩略图
- 是否区分头像、内容图、背景图
- 上传失败是否有提示
- 上传中是否禁止重复点击

## 二、图片压缩策略

推荐默认：

- 头像压缩到 512×512 或 1024×1024
- 内容图按最长边 1600 或 2048 压缩
- 使用 JPEG / WebP 等常见格式
- 上传前检查格式和体积

需要根据具体项目确认：

- 是否保留透明背景
- 是否允许 GIF
- 是否允许 HEIC
- 是否需要转换格式
- 是否需要多尺寸缩略图
- 图片质量参数是多少

## 三、头像上传规则

头像上传需要同时检查 UI 和媒体规则。

检查点：

- 裁剪框是否与展示形状一致
- 圆形头像是否使用圆形裁剪预览
- 方形头像是否使用方形裁剪预览
- 上传后是否立即刷新
- 刷新页面后是否仍然正确
- 不同页面是否使用同一图片源
- 旧头像缓存是否清除
- 默认头像是否统一
- 上传失败是否恢复原状态

## 四、图片存储规则

必须明确：

- 图片存在数据库还是对象存储
- 数据库存储图片 URL 还是文件本体
- 文件路径如何生成
- 文件名是否唯一
- 是否按 userId / postId / petId 分类
- 是否允许覆盖旧文件
- 是否需要删除旧文件
- 单用户最多上传多少
- 单条内容最多上传多少张图
- 是否需要清理废弃图片
- 删除内容后图片是否一起删除

## 五、上传错误提示

常见错误包括：

- 格式不支持
- 图片太大
- 网络失败
- 上传超时
- 权限不足
- 存储桶配置错误
- 文件路径错误

错误提示不能只在控制台显示，用户界面也要有反馈。

## 六、图片刷新规则

图片上传成功后，需要检查：

- 当前页面是否立即更新
- 返回上一级后是否更新
- 列表页是否更新
- 详情页是否更新
- 评论区是否更新
- 消息页是否更新
- 重新打开 App 后是否更新
- 是否存在缓存旧图问题

## 七、数据库字段规则

涉及图片时，需要确认字段含义：

- avatarUrl
- imageUrl
- coverUrl
- thumbnailUrl
- filePath
- storagePath
- publicUrl
- userId
- ownerId

不要把文件路径、公开 URL、缩略图 URL 混用。

## 八、图片安全与权限

需要检查：

- 用户是否只能上传自己的图片
- 用户是否只能删除自己的图片
- 存储桶权限是否正确
- 公共图片和私有图片是否区分
- 密钥是否暴露到前端
- 上传路径是否可被恶意覆盖
