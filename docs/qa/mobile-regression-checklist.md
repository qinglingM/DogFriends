# 真机上线前回归清单

本清单用于正式上线前的真机回归。

原则：

1. 先测核心主流程，再测边界场景。
2. 每个写入型操作都要验证：是否成功、失败是否可感知、刷新后是否一致。
3. 每个用户身份展示都要验证：头像、昵称、跳转是否正确。
4. 每个弱网场景都要验证：是否回滚、是否重复提交、是否出现假成功。

## 一、推荐回归顺序

| 顺序 | 模块 | 原因 |
|---|---|---|
| 1 | 登录、资料、发帖、评论 | 核心社交流程，最先保证可用 |
| 2 | 探索首页、新增地点、地点反馈 | 多步写入最多，最容易出脏数据 |
| 3 | 关注、点赞、收藏、我的点赞 | 最容易出现计数漂移 |
| 4 | 遛狗开始、打卡、结束、历史 | 最容易出现假成功 |
| 5 | 弱网专项回归 | 专门抓失败提示、回滚、重复提交 |

## 二、核心流程回归

| 模块 | 回归项 | 操作路径 | 预期结果 |
|---|---|---|---|
| 登录 | 首次登录 | 启动 App -> 登录 | 能正常进入首页，无白屏、无卡死 |
| 登录 | 重启保活 | 登录后杀掉 App 重开 | 登录状态仍保留 |
| 登录 | 弱网登录 | 弱网/断网下登录 | 有明确失败提示，不会假登录成功 |
| 个人资料 | 修改资料同步 | 改昵称/头像/签名后返回主页 | 主页、帖子、评论等相关展示同步更新 |
| 个人资料 | 昵称缺失拦截 | 清空昵称后尝试发帖 | 不能发布，并提示先完善昵称 |
| 狗狗资料 | 新增狗狗 | 个人页 -> 新增狗狗 | 能成功保存，列表立即可见 |
| 狗狗资料 | 编辑狗狗 | 修改名字/品种/体重 | 保存成功，刷新后仍正确 |
| 狗狗资料 | 体重上限 | 输入超大数字 | 不会超过设定上限，显示正常 |
| 发帖 | 正常发帖 | 广场 -> 发帖 | 成功发布，广场列表和详情页都能看到 |
| 发帖 | 防重复提交 | 连点发布按钮 | 只发布一次，按钮有 loading |
| 发帖 | 失败保留输入 | 断网后发布 | 有失败提示，已输入内容不要丢失 |
| 评论 | 发表评论 | 帖子详情 -> 输入评论 -> 发送 | 评论成功出现，数量同步增加 |
| 评论 | 回复评论 | 点回复 -> 输入 -> 发送 | 回复成功出现，归属正确 |
| 评论 | 评论失败 | 断网/弱网下发送评论 | 有失败提示，不留半套评论数据 |
| 评论输入 | 键盘跟随 | 帖子详情 -> 点评论框/回复 | 输入框始终在键盘上方，不被遮挡 |
| 评论输入 | 超长提示 | 评论输入超过限制 | 不显示持续字数计数，超限时出现短暂提示 |

## 三、探索模块回归

| 模块 | 回归项 | 操作路径 | 预期结果 |
|---|---|---|---|
| 探索首页 | 地点列表加载 | 去玩 Tab 首页 | 能正常展示地点，不白屏 |
| 探索首页 | 收藏地点 | 点地点收藏/取消收藏 | 状态正确，刷新后仍一致 |
| 新增地点 | 正常新增 | 去玩 -> 新增地点 -> 填完整发布 | 成功创建，成功页和列表数据一致 |
| 新增地点 | 防重复提交 | 连点发布地点 | 只提交一次 |
| 新增地点 | 城市必填 | 城市缺失时发布 | 不会写成假城市，直接拦截 |
| 地点反馈 | 正常反馈 | 地点详情 -> 我来反馈 -> 发布 | 成功发布，地点详情立即可见 |
| 地点反馈 | 每日限制 | 同一天重复反馈同一地点 | 有明确提示，不能重复提交 |
| 地点反馈 | 防重复提交 | 连点发布反馈 | 只提交一次 |

## 四、身份与资料绑定回归

| 模块 | 回归项 | 操作路径 | 预期结果 |
|---|---|---|---|
| 用户身份 | 头像展示 | 看帖子作者、评论作者、地点验证者 | 有真实头像优先显示真实头像 |
| 用户身份 | 缺资料兜底 | 找 profile 不完整的测试账号 | 不应频繁出现“未知用户”，兜底展示稳定 |
| 用户身份 | 个人主页跳转 | 从帖子/评论/验证点头像或名字进入主页 | 进入正确用户主页，不串号 |
| 关注关系 | 关注/取关 | 进入他人主页 -> 关注/取关 | 数量变化正确，刷新后仍一致 |
| 点赞页 | 我的点赞 | 个人页 -> 我的点赞 | 列表正常，作者信息正常，点作者能跳正确主页 |

## 五、遛狗模块回归

| 模块 | 回归项 | 操作路径 | 预期结果 |
|---|---|---|---|
| 遛狗 | 开始遛狗 | 选择狗狗 -> 开始 | 正常进入遛狗流程 |
| 遛狗 | 结束遛狗成功 | 完成遛狗 -> 保存打卡 | 成功进入结果页，历史记录可见 |
| 遛狗 | 结束遛狗失败 | 断网后结束遛狗 | 不应假成功，应提示保存失败 |
| 遛狗 | 统计更新 | 完成一次遛狗后看狗狗统计 | 次数/距离/时长正确增加 |

## 六、弱网专项回归

| 模块 | 回归项 | 操作路径 | 预期结果 |
|---|---|---|---|
| 全局写入操作 | 发帖、评论、点赞、收藏、新增地点、反馈、结束遛狗 | 弱网/断网下重复操作 | 失败要有提示，成功后刷新一致 |
| 刷新一致性 | 关键操作后重进验证 | 操作后下拉刷新/重进 App | 数据不丢、不重复、不变形 |
| 控制台 | 运行时错误 | 全流程操作一遍 | 不应出现明显红屏、白屏、致命报错 |

## 七、重点高风险项

| 优先级 | 项目 | 为什么重点测 |
|---|---|---|
| P0 | 结束遛狗 | 之前存在假成功风险 |
| P1 | 新增地点 / 地点反馈 | 多步写入，最容易留半套数据 |
| P1 | 评论发送 / 回复 | 既有输入交互，又有计数同步 |
| P1 | 点赞 / 收藏 / 关注 | 最容易出现弱网漂数 |
| P1 | 用户主页跳转 | 最容易暴露身份绑定问题 |

## 八、回归记录建议

| 字段 | 建议填写内容 |
|---|---|
| 测试日期 | 例如：2026-07-01 |
| 测试设备 | 例如：iPhone 17 Pro / Android 机型 |
| 测试环境 | Wi-Fi / 4G / 弱网 / 断网 |
| 测试账号 | 测试账号标识 |
| 结果 | 通过 / 失败 |
| 问题记录 | 失败路径、现象、截图、是否可复现 |

## 九、上线前最低标准

| 标准 | 要求 |
|---|---|
| 核心流程 | 登录、发帖、评论、探索、遛狗必须可用 |
| 写入结果 | 不允许假成功 |
| 重复提交 | 关键写入不允许重复数据 |
| 身份展示 | 不允许明显错绑或频繁未知用户 |
| 刷新一致性 | 刷新后数据不丢不漂 |
| 真机体验 | 键盘、安全区、输入框、底部按钮正常 |
