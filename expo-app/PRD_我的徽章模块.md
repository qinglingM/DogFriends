# PRD：我的徽章模块

## 1. 模块定位

### 1.1 模块名称

**我的徽章**

### 1.2 所属位置

位于 APP 底部 Tab「我的」页面中。

页面结构建议：

```text
我的
  ├─ 用户信息
  ├─ 我的收藏
  ├─ 我的贡献记录
  ├─ 我的徽章
  └─ 设置
```

### 1.3 模块目标

「我的徽章」用于记录和展示用户在宠物友好地点共建中的贡献成就，鼓励用户持续发布真实地点、真实验证、真实照片和真实避坑信息。

### 1.4 用户价值

用户可以清楚看到：

1. 我已经获得了哪些徽章；
2. 我还差多少可以获得下一枚徽章；
3. 我的哪些真实贡献正在帮助其他狗主人；
4. 我可以通过什么行为继续解锁徽章。

---

## 2. 设计原则

### 2.1 徽章奖励真实贡献

徽章只奖励对其他狗主人有实际帮助的行为，包括：

* 新增真实宠物友好地点；
* 发布真实到访验证；
* 补充宠物友好规则；
* 提交避坑信息；
* 上传有效地点照片；
* 获得其他用户“有用”反馈；
* 补充大型犬、雨天、室内可进等关键场景信息。

### 2.2 徽章用于激励，不制造打扰

徽章信息集中展示在「我的」页面和「我的徽章」详情页中。
不通过消息通知频繁打扰用户。

### 2.3 徽章进度必须可理解

每个未解锁徽章都需要展示当前进度，例如：

```text
2/5
还差 3 次验证
```

用户需要一眼知道自己为什么还没有获得，以及下一步可以做什么。

## 3. 页面结构

本模块包含 3 个页面/区域：

```text
我的页面 · 我的徽章模块
  └─ 点击「查看全部」
      └─ 我的徽章页
          └─ 点击单个徽章
              └─ 徽章详情页
```

---

## 4. 我的页面中的徽章模块

### 4.1 所在位置

位于「我的贡献记录」下方。

页面顺序：

```text
用户信息
我的收藏
我的贡献记录
我的徽章
设置入口
```

### 4.2 模块标题

左侧标题：

```text
我的徽章
```

右侧入口：

```text
查看全部
```

点击「查看全部」进入「我的徽章页」。

### 4.3 首页展示内容

在「我的」页面中，只展示用户最近获得或最有代表性的 3 枚徽章。

#### 展示方式

横向展示 3 个徽章。

每个徽章包含：

| 元素 | 说明 |
| --- | --- |
| 徽章图标 | 圆形或盾牌形图标 |
| 徽章名称 | 例如“城市发现者” |
| 获得状态 | 已获得 / 未解锁 |
| 简短说明 | 例如“新增 1 个地点” |

### 4.4 点击行为

| 点击对象 | 结果 |
| --- | --- |
| 点击「查看全部」 | 进入我的徽章页 |
| 点击某个徽章 | 进入该徽章详情页 |

---

## 5. 我的徽章页

### 5.1 页面目标

展示用户所有徽章，以及每个徽章的获得状态和进度。

### 5.2 页面入口

入口：

```text
我的 → 我的徽章 → 查看全部
```

### 5.3 页面顶部

顶部导航栏：

| 元素 | 位置 |
| --- | --- |
| 返回按钮 | 左上角 |
| 页面标题「我的徽章」 | 居中 |
| 无额外操作按钮 | 右侧为空 |

### 5.4 页面头部信息

页面顶部展示一句总结文案：

```text
你的真实分享，正在帮助更多狗狗出门不踩雷
```

下方展示用户已获得徽章数量：

```text
已获得 3/9 枚徽章
```

### 5.5 徽章展示布局

徽章采用九宫格展示。

#### 布局规则

| 规则 | 内容 |
| --- | --- |
| 每行数量 | 3 个徽章 |
| 默认展示 | 3 行，共 9 个徽章 |
| 已获得徽章 | 彩色图标 |
| 未获得徽章 | 灰色图标 |
| 每个徽章下方 | 展示徽章名称 |
| 未获得徽章 | 展示进度，如 2/5 |

示例：

```text
城市发现者      城市探索家      可靠验证官
已获得          2/5             3/5

避坑提醒员      现场记录员      有用分享家
1/3             已获得          6/10

大型犬友好官    雨天救星        城市带狗向导
1/3             2/3             24/50
```

### 5.6 徽章状态

每个徽章只有两种状态：

| 状态 | 展示 |
| --- | --- |
| 已获得 | 彩色徽章 + 获得状态 |
| 未获得 | 灰色徽章 + 当前进度 |

### 5.7 点击行为

点击任意徽章，进入该徽章详情页。

## 6. 徽章详情页

### 6.1 页面目标

让用户理解某一枚徽章的意义、获得条件、当前进度，以及下一步可以做什么。

### 6.2 页面入口

入口：

```text
我的徽章页 → 点击任意徽章
```

### 6.3 页面结构

从上到下：

| 区域 | 内容 |
| --- | --- |
| 顶部导航栏 | 返回按钮 + 页面标题 |
| 徽章大图 | 居中展示徽章图标 |
| 徽章名称 | 例如“避坑提醒员” |
| 徽章说明 | 这枚徽章代表什么 |
| 获得条件 | 明确告诉用户如何获得 |
| 当前进度 | 展示进度条和数字 |
| 下一步行动 | 根据徽章类型展示行动按钮 |

### 6.4 已获得徽章详情

如果用户已获得该徽章，展示：

| 字段 | 示例 |
| --- | --- |
| 徽章名称 | 城市发现者 |
| 徽章说明 | 你发现了第一个可以带狗去的地方 |
| 获得条件 | 新增 1 个宠物友好地点 |
| 获得时间 | 2026.06.08 |
| 当前状态 | 已获得 |

#### 已获得文案示例

```text
你已经获得「城市发现者」

你发现了第一个可以带狗去的地方，
让更多狗狗多了一个出门选择。
```

底部按钮：

```text
继续贡献
```

点击后进入「去玩」Tab。

### 6.5 未获得徽章详情

如果用户未获得该徽章，展示：

| 字段 | 示例 |
| --- | --- |
| 徽章名称 | 可靠验证官 |
| 徽章说明 | 你的真实到访经验，能帮大家判断地点是否靠谱 |
| 获得条件 | 发布 5 条地点验证 |
| 当前进度 | 3/5 |
| 进度条 | 当前完成 60% |
| 下一步提示 | 还差 2 条验证 |

#### 未获得文案示例

```text
还差 2 条验证，获得「可靠验证官」

去过宠物友好地点后，
补充真实规则，可以帮助其他狗主人少踩雷。
```

底部按钮：

```text
去验证地点
```

点击后进入「去玩」Tab。

## 7. 第一版徽章清单

第一版共 9 枚徽章。

### 7.1 城市发现者

#### 维度

新增地点。

#### 获得条件

```text
新增 1 个宠物友好地点
```

#### 徽章说明

```text
你发现了一个可以带狗去的地方，让更多狗狗多了一个出门选择。
```

#### 下一步按钮

```text
去新增地点
```

### 7.2 城市探索家

#### 维度

持续新增地点。

#### 获得条件

```text

```

#### 徽章说明

```text
你持续发现城市里的宠物友好地点，正在帮大家打开更多带狗出门的选择。
```

#### 下一步按钮

```text
去新增地点
```

### 7.3 可靠验证官

#### 维度

真实到访验证。

#### 获得条件

```text
发布 5 条地点验证
```

#### 有效验证包括

* 我带狗成功去了；
* 只能坐户外；
* 可以进室内；
* 大型犬可以；
* 大型犬不行；
* 不让进了；
* 需要牵绳；
* 有水碗；
* 店员友好；
* 空间宽敞。

#### 徽章说明

```text
你的真实到访经验，能帮其他狗主人判断这个地方到底适不适合去。
```

#### 下一步按钮

```text
去验证地点
```

### 7.4 避坑提醒员

#### 维度

规则变化 / 不可带反馈。

#### 获得条件

```text

```

#### 有效反馈包括

* 只能户外，不能室内；
* 大型犬不能进；
* 周末不让进；
* 地址不准确；
* 原信息不准确。

#### 徽章说明

```text
你帮其他狗主人减少白跑一趟的可能。
```


### 7.5 现场记录员

#### 获得条件


#### 有效照片包括

* 门店入口；
* 户外座位；
* 室内环境；
* 宠物友好标识；
* 狗狗到访照片；

#### 徽章说明

```text
你的照片让大家更直观地判断这个地方是否适合带狗去。
```

```text
去补充照片
```

```text
其他狗主人觉得你的分享真的帮到了他们。
```


#### 获得条件

* 大型犬需嘴套；

```text
大型犬出门更容易踩雷，你的补充能帮大型犬家庭提前判断。

```

#### 获得条件

```text
提交 3 个室内可进、下雨可去或有遮挡的地点信息
```

#### 有效信息包括

* 室内可进；
* 有遮挡；
* 雨天可去；
* 不怕下雨；

#### 徽章说明

```text
下雨天也能带狗出门，你帮大家找到了更安心的选择。

```

| 行为 | 计入帮助 |

| 用户新增的地点被其他用户验证 | 计入 |
| 用户提交的避坑反馈被点有用 | 计入 |

#### 徽章说明

```text
你正在让这座城市对狗狗和狗主人更友好。
```

#### 下一步按钮

```text
查看贡献记录
```

## 8. 徽章数据计算规则

### 8.1 新增地点数

统计用户通过「新增我去过的地点」发布的地点数量。

### 8.2 地点验证数

统计用户在地点详情页点击「我去过，更新信息」后发布的验证数量。

### 8.3 避坑反馈数

统计用户提交的负向或规则变化类信息，包括：

* 不让进了；
* 大型犬不行；
* 只能户外；
* 地址不准确；
* 信息不准。

### 8.4 有效照片数

统计用户上传的地点相关照片数量。

### 8.5 有用数

统计其他用户对用户贡献内容点击「有用」的次数。

### 8.6 帮助人数

第一版帮助人数由以下数据合并计算：

```text
帮助人数 = 有用数 + 新增地点被收藏数 + 新增地点被其他用户验证数 + 避坑反馈被点有用数
```

同一用户对同一内容的重复行为只计一次。

## 9. 徽章墙空状态调整

### 9.1 核心原则

* **徽章墙永远展示全部徽章**，即第一版设计的 9 枚徽章。
* 用户未获得的徽章显示为**灰度图标**。
* 用户已获得的徽章显示为**彩色图标**。
* 不再出现“空状态文案”或“去新增地点”按钮。
* 空状态不再依赖用户是否获得过徽章。

### 9.2 我的徽章页

顶部导航栏：

* 左：返回按钮
* 中：页面标题「我的徽章」

页面头部：

* 文案：`你的真实分享，正在帮助更多狗狗出门不踩雷`
* 已获得徽章数量：`已获得 0/9 枚徽章`（可为 0）

徽章展示：

* 采用九宫格
* 每行 3 个徽章
* 已获得：彩色图标
* 未获得：灰度图标
* 下方显示徽章名称
* 未获得徽章显示当前进度，如 `0/5`

### 9.3 点击行为

* 点击已获得或未获得徽章都进入该徽章的详情页
* 徽章详情页仍展示徽章名称、说明、获得条件、当前进度、下一步行动提示
* 当用户徽章数量为 0 时，进度条全部显示为 0%

## 10. 徽章解锁反馈

### 10.1 解锁时机

当用户完成某个徽章条件后，在对应动作完成页展示轻提示。

例如用户发布第 5 条验证后，弹出：

```text
获得新徽章：可靠验证官
```

### 10.2 展示方式

使用页面内轻弹层，不使用消息通知。

弹层内容：

| 元素 | 内容 |
| --- | --- |
| 徽章图标 | 彩色徽章 |
| 标题 | 获得新徽章 |
| 徽章名称 | 可靠验证官 |
| 说明 | 你的真实验证正在帮助更多狗主人 |
| 按钮 1 | 查看徽章 |
| 按钮 2 | 知道了 |

### 10.3 点击行为

| 按钮 | 结果 |
| --- | --- |
| 查看徽章 | 进入该徽章详情页 |
| 知道了 | 关闭弹层，回到当前页面 |

## 11. 与其他模块的关系

### 11.1 与「去玩」模块

徽章数据主要来自「去玩」模块中的行为：

* 新增地点；
* 更新地点信息；
* 上传照片；
* 点有用；
* 收藏地点；
* 查看地点详情。

### 11.2 与「我的贡献记录」

「我的贡献记录」展示用户具体做过哪些贡献。
「我的徽章」展示这些贡献沉淀出来的成就。

用户可以从徽章详情页跳转到对应贡献动作。

### 11.3 与「我的收藏」

收藏行为会影响「城市带狗向导」的帮助人数统计，但收藏模块本身不产生徽章页面入口。

## 13. 最终产品闭环

```text
用户新增真实地点 / 发布验证 / 上传照片 / 提交避坑信息
→ 系统累计贡献数据
→ 用户解锁徽章
→ 用户在「我的徽章」看到成就和下一步进度
→ 用户继续贡献真实信息
→ 去玩地点库质量提升
```

---

## 14. 最终定义

「我的徽章」不是装饰功能，而是一个**真实贡献激励模块**。

它奖励的不是用户活跃，而是用户对其他狗主人的实际帮助：

> 发现更多能带狗去的地方，验证这些地方是否真的友好，补充规则，减少踩雷，让这座城市对狗狗更友好。
