# PRD：去玩 Tab · 宠物友好地点模块

> 本期产品原则：**这不是平台认证的宠物友好地点目录，而是狗主人之间共享带狗出门经验的城市地点列表。** 平台不做事前审核，只提供地点展示、用户验证、信息更新、纠错反馈与风险提示。

---

## 1. 模块定位

### 1.1 模块名称

**去玩**

### 1.2 所属位置

APP 底部四个一级 Tab 中的一个：

| Tab | 作用            |
| --- | ------------- |
| 遛狗  | 记录遛狗轨迹、时长、距离  |
| 去玩  | 发现城市里适合带狗去的地方 |
| 记录  | 查看历史遛狗/出行记录   |
| 我的  | 用户资料、狗狗资料、设置  |

当前 PRD 只描述 **「去玩」Tab**。

### 1.3 一句话定义

**「去玩」是一个帮助狗主人发现并共建城市内宠物友好地点的列表型功能模块。**

用户可以查看城市里有哪些公园、咖啡店、餐厅适合带狗去，查看每个地点的宠物规则、最近验证记录，也可以把自己真实去过的宠物友好地点发布出来，让其他狗主人验证、补充或纠错。

### 1.4 核心用户场景

| 场景    | 用户问题               |
| ----- | ------------------ |
| 周末出门  | 这个城市有哪些地方适合带狗去？    |
| 想喝咖啡  | 哪些咖啡店可以带狗？         |
| 想逛公园  | 哪些公园对狗友好？          |
| 想带狗吃饭 | 哪些餐厅能带狗？           |
| 发现新地点 | 我真实去过一个宠物友好地点，想分享给大家 |
| 信息更新  | 我去了之后发现规则变了，想更新信息  |
| 信息纠错  | 我看到的信息和实际不符，想反馈    |

## 3. 信息源与数据原则

### 3.1 地点底库来源

地点底库可以来自地图 POI 服务（高德、腾讯位置服务、百度地图等）的关键字、周边、区域、ID 检索能力，用于获得地点名称、地址、电话、坐标、分类等基础信息。POI 不能直接判断“是否宠物友好”。

### 3.2 宠物友好信息来源（共建）

| 来源        | 用途          |
| --------- | ----------- |
| 平台运营预置    | 初始种子地点      |
| C 端用户新增   | 补充新地点（直接发布） |
| C 端用户验证   | 更新地点规则、新增反馈 |
| 用户上传图片    | 提供现场证据      |
| 用户「信息不准」反馈 | 触发地点状态变化    |

### 3.3 信息真实性原则

> **平台不对单条信息做事前审核，但通过用户多人反馈形成可信度。**

地点是否宠物友好，不能只存一个“是/否”，必须存规则。

核心规则字段：

| 字段     | 示例                        |
| ------ | ------------------------- |
| 是否可带狗  | 可带 / 仅户外 / 不稳定 / 不建议      |
| 可进入区域  | 室内 / 室外 / 仅露台 / 仅门口       |
| 适合狗狗类型 | 小型犬 / 中型犬 / 大型犬 / 所有体型    |
| 行为要求   | 牵绳 / 嘴套 / 不上椅 / 不吠叫       |
| 配套设施   | 水碗 / 宠物餐 / 拾便袋 / 遮阳区      |
| 最近更新时间 | 2 天前 / 7 天前 / 30 天前       |
| 用户共建状态 | 用户提交 / 最近有人去过 / 多人验证 / 信息有争议 / 规则可能变化 / 不建议前往 |

### 3.4 平台不承诺

* 这个地点一定能带狗；
* 这个信息一定永久有效；
* 这个规则一定准确。

平台提供：地点展示、用户验证、信息更新、纠错反馈、最近验证记录、风险提示。

## 4. 页面结构总览

本模块共包含 8 个核心页面/状态：

```text
去玩首页
  ├─ 地点列表
  ├─ 搜索与筛选
  ├─ 新增我去过的地点入口
  └─ 点击地点进入详情

地点详情页
  ├─ 宠物规则
  ├─ 狗主人验证模块（新增）
  ├─ 信息免责声明
  ├─ 收藏 / 导航 / 电话
  └─ 我去过，更新信息

更新信息页（我去过）
  └─ 直接发布为狗主人验证

新增地点流程（不经过审核）
  ├─ 填写地点基本信息页（1/3）
  ├─ 填写宠物友好规则页（2/3）
  ├─ 上传图片与补充信息页（3/3）
  ├─ 发布成功页
  └─ 我的贡献 / 消息提醒
```

> ⚠️ 原「选择新增方式页」已删除，新增地点入口直接进入「填写地点基本信息页」。

## 5. 页面一：去玩首页

### 5.1 页面目标

帮助用户快速浏览当前城市内适合带狗去的地点，并提供新增宠物友好地点入口。

### 5.2 页面位置

底部 Tab 栏第二个 Tab：**去玩**

### 5.3 页面结构

从上到下：

| 区域     | 内容                       |
| ------ | ------------------------ |
| 顶部状态栏  | 系统时间、信号、电量               |
| 城市区域   | 当前城市，如“上海 ∨”             |
| 顶部操作区  | 搜索 icon、通知 icon          |
| 搜索框    | 搜索地点 / 区域 / 咖啡店 / 公园     |
| 页面标题   | 去玩                       |
| 副标题    | 带上狗，一起探索城市               |
| 分类入口   | 全部、公园、咖啡、餐厅              |
| 场景筛选   | 多人验证、室内可进、室外友好、大型犬友好     |
| 地点列表   | 宠物友好地点卡片（带共建状态）          |
| 新增地点按钮 | 固定在列表底部                  |
| 底部 Tab | 遛狗、去玩、记录、消息/我的           |

### 5.4 顶部区域

* **城市切换**：左上角“上海 ∨”，点击进入「选择城市页」。
* **搜索按钮**：右上角第一个 icon，进入搜索页。
* **通知按钮**：右上角第二个 icon，进入消息页。

### 5.5 搜索框

占位文案：`搜索地点 / 区域 / 咖啡店 / 公园`，点击进入搜索页。

### 5.6 分类入口

icon 分类：**全部 / 公园 / 咖啡 / 餐厅**，默认“全部”。

### 5.7 场景筛选

横向 Chip：

| 筛选项   | 含义             |
| ----- | -------------- |
| 多人验证  | 至少 3 位狗主人验证可带狗 |
| 室内可进  | 狗可以进入室内        |
| 室外友好  | 有露台/户外/门口座位    |
| 大型犬友好 | 用户反馈大型犬可以去     |

> 原“已验证”筛选已替换为“多人验证”，对应新的共建状态体系。

### 5.8 地点列表卡片

每张卡片展示一个地点。

#### 卡片字段

| 字段     | 示例                  |
| ------ | ------------------- |
| 地点图片   | 卡片左上                |
| 地点名称   | BLOOM Coffee        |
| 类型+区域+距离 | 咖啡店 · 徐汇区 · 1.8km |
| 宠物状态   | 最近有人去过              |
| 信息来源   | 3 位狗主人验证            |
| 最近更新   | 2 天前更新              |
| 关键规则   | 仅户外 / 有水碗 / 小型犬友好   |
| 收藏按钮   | 卡片右上角               |

#### 示例

```text
BLOOM Coffee
咖啡店 · 徐汇区 · 1.8km
最近有人去过
3 位狗主人验证 · 2 天前更新
仅户外｜有水碗｜小型犬友好
```

新提交地点的卡片：

```text
爪爪咖啡 Paw Cafe
咖啡店 · 徐汇区 · 1.2km
用户提交
等待更多狗主人验证
室外可进｜大型犬友好｜有水碗
```

#### 点击行为

| 点击区域       | 结果        |
| ---------- | --------- |
| 点击卡片主体     | 进入地点详情页   |
| 点击收藏 icon  | 收藏/取消收藏   |
| 点击分类/场景筛选  | 刷新列表      |
| 点击新增地点按钮   | 进入新增基本信息页 |

### 5.9 新增宠物友好地点按钮

* **位置**：去玩首页底部，靠近底部 Tab。
* **样式**：绿色实心按钮。
* **文案**：`+ 新增我去过的地点`
* **点击结果**：直接进入「填写地点基本信息页」（无中间“选择方式页”）。

## 6. 页面二：填写地点基本信息页（1/3）

### 6.1 页面目标

收集新增地点的基础信息。

### 6.2 页面入口

「去玩首页」点击 `+ 新增我去过的地点`。

### 6.3 页面标题

```text
新增地点 · 基本信息
```

右侧进度：`1/3`

### 6.4 顶部温馨提示

```text
请只提交你真实去过、确认可以带狗的地点。
信息会直接发布，请尽量填写准确。
```

### 6.5 表单字段

| 字段   | 必填 | 占位/示例                |
| ---- | -- | -------------------- |
| 地点名称 | 是  | 请输入地点名称（2–40 字）      |
| 地点类型 | 是  | 公园 / 咖啡 / 餐厅 / 商场 / 其他（单选） |
| 所在城市 | 是  | 默认当前城市，可选择           |
| 所在区域 | 是  | 选择区/县                |
| 详细地址 | 是  | 例：某某路 123 号          |
| 地图位置 | 选填 | 在地图上选位置（辅助）          |
| 联系电话 | 选填 | 用户出发前电话确认            |
| 营业时间 | 选填 | 例：10:00 - 22:00      |

### 6.6 底部按钮

`下一步`，必填项填完后变绿色可点击。点击进入「填写宠物友好规则页」。

## 7. 页面三：填写宠物友好规则页（2/3）

### 7.1 页面目标

提交该地点的宠物友好规则。

### 7.2 页面标题

```text
宠物友好规则
```

右侧进度：`2/3`

### 7.3 表单字段

* **宠物可进入区域（必填，单选）**：室内可进 / 室外可进 / 仅露台或户外 / 指定区域可进 / 不可进入
* **适合的狗狗（必填）**：小型犬友好 / 中型犬友好 / 大型犬友好 / 所有体型
* **行为要求（多选，选填）**：需要牵绳 / 需要嘴套 / 不能上椅·沙发 / 不能吠叫 / 不影响其他客人 / 需要狗狗安静待着
* **配套设施（多选，选填）**：有水碗 / 有宠物餐 / 有狗狗零食 / 有便袋 / 有遮阳区 / 有围栏 / 有草坪 / 其他

### 7.4 底部按钮

* 左：`上一步` → 返回基本信息页
* 右：`下一步` → 进入「上传图片与补充信息页」

## 8. 页面四：上传图片与补充信息页（3/3）

### 8.1 页面目标

让用户上传证据图片并补充真实体验，然后**直接发布**。

### 8.2 页面标题

```text
补充信息 & 发布
```

右侧进度：`3/3`

### 8.3 表单字段

| 字段        | 必填           | 说明                                     |
| --------- | ------------ | -------------------------------------- |
| 补充说明      | 选填           | 占位：`分享一些你对这个地方的真实体验吧～`                 |
| 上传图片      | 至少 1 张       | 门店入口 / 室内环境 / 户外座位 / 宠物友好标识 / 狗狗到访 等   |
| 推荐原因标签    | 选填           | 自己去过 / 朋友推荐 / 网络看到 / 其他                |

### 8.4 底部按钮

按钮文案：

```text
发布地点
```

#### 可点击条件

1. 基本信息必填项已完成

> 不再有“提交审核”按钮，不再有“等待审核”状态。

## 10. 页面六：地点详情页

### 10.1 页面目标

展示地点完整信息，让用户判断是否带狗前往，并提供查看用户验证、更新信息、纠错反馈等能力。

### 10.2 页面结构（自上而下）

```text
地点图片
地点名称（+收藏）
地点类型 · 区域 · 距离
宠物友好状态（含信息免责声明）
宠物规则（可进入区域、行为要求等）
适合的狗狗
狗主人验证（新增）
信息不准反馈入口
底部操作按钮（导航 / 电话 / 我去过更新信息）
```

### 10.3 宠物友好状态区

* 顶部 Banner 展示地点的共建状态（如“最近有人去过”、“多人验证”、“信息有争议”、“用户提交”）。
* Banner 下方一行浅色提示：

```text
信息由狗主人共同更新，可能随时间变化，出发前建议电话确认。
```

#### 顶部风险提示

| 触发条件               | 顶部提示                       |
| ------------------ | -------------------------- |
| 状态为「信息有争议」         | 该地点信息有争议，建议出发前电话确认         |
| 状态为「规则可能变化」/近期有人被拒 | 近期有人反馈被拒，建议出发前确认            |
| 状态为「用户提交」（新发布）     | 这个地点由用户新增，暂时还没有多人验证。如果你去过，也可以帮大家更新信息 |

### 10.4 狗主人验证模块（核心新增）

#### 模块名称

```text
狗主人验证
```

> 不叫“点评”、不叫“评价”——核心是验证信息，不是吃喝感受。

#### 每条验证卡片字段

| 字段    | 示例                  |
| ----- | ------------------- |
| 用户头像  | 默认头像 / 狗狗头像         |
| 用户昵称  | 豆豆妈妈                |
| 时间    | 2 天前                |
| 到访结果  | 我带狗成功去了             |
| 狗狗类型  | 小型犬 / 大型犬 / 柯基      |
| 关键标签  | 只能户外 / 有水碗 / 店员友好   |
| 文字补充  | “店员很好，给了水碗，露台位置宽敞。” |
| 图片    | 用户上传的现场照片           |
| 有用按钮  | 「有用」+ 当前票数          |
| 信息不准  | 信息不准（轻量纠错入口）        |

#### 卡片示例

```text
狗主人验证
─────────────────────────────
豆豆妈妈 · 2 天前
我带狗成功去了｜小型犬
仅户外｜有水碗｜店员友好
店员很好，给了水碗。露台位置比较宽，狗狗可以安静待着。
[照片] [照片]
👍 有用 12        ⚠ 信息不准
```

#### 展示规则

| 情况        | 展示方式                            |
| --------- | ------------------------------- |
| 有多条验证     | 默认展示最近 3 条                      |
| 超过 3 条    | 显示「查看全部验证」入口                    |
| 没有任何验证    | 显示空状态                           |
| 信息冲突      | 模块顶部「该地点信息有争议」标签                |
| 近期有人反馈被拒  | 模块顶部「近期有人反馈被拒，建议出发前确认」          |

#### 空状态

```text
还没有狗主人验证过这个地点
如果你去过，可以帮大家补充信息
```

底部按钮：

```text
我去过，更新信息
```

### 10.5 底部操作

* `我去过，更新信息`（次要按钮，全宽）

## 13. 地点共建状态体系

> 完全取代原「审核中 / 已通过 / 未通过」体系。

### 13.1 状态定义

| 状态     | 含义                 | 触发条件                  |
| ------ | ------------------ | --------------------- |
| 用户提交   | 该地点由用户新增，平台不背书真实性  | 新地点发布后，且尚无其他用户验证      |
| 最近有人去过 | 近 30 天有人验证成功带狗到访   | 至少 1 条成功到访验证          |
| 多人验证   | 多个用户反馈可带狗          | ≥3 位不同用户提交“成功到访”验证    |
| 信息有争议  | 用户反馈互相冲突           | 同时存在成功到访和被拒反馈，或多次信息不准 |
| 规则可能变化 | 最近有人反馈被拒或规则变更      | 近 30 天出现 ≥1 条“不让进了”反馈 |
| 不建议前往  | 多个用户反馈不可带狗或体验差     | 连续 ≥3 条不可带反馈，且无近期成功反馈 |

### 13.2 状态颜色建议

| 状态     | 主色调       |
| ------ | --------- |
| 用户提交   | 中性灰       |
| 最近有人去过 | 主品牌绿      |
| 多人验证   | 主品牌绿（深一档） |
| 信息有争议  | 警示橙       |
| 规则可能变化 | 警示橙       |
| 不建议前往  | 警示红       |

### 13.3 状态在卡片/详情页的展示

* **列表卡片**：单行展示，伴随“信息来源 · 最近更新”一行说明。
* **详情页**：顶部 Banner 展示，并在状态为争议/规则可能变化时附加风险提示文案。

## 22. 最终 PRD 结论

本模块不是“平台认证宠物友好地点”，而是：

> **狗主人之间共享带狗出门经验的城市地点列表。**

平台不承诺：

* 这个地点一定能带狗；
* 这个信息一定永久有效；
* 这个规则一定准确。

平台提供：

* 地点展示；
* 用户验证；
* 信息更新；
* 纠错反馈；
* 最近验证记录；
* 风险提示。

本期最重要的体验是：

1. **打开去玩，能看到当前城市适合带狗去的地点，并能看到信息来源；**
2. **点进地点，能看懂狗能不能去、能进哪里、最近其他狗主人怎么说；**
3. **用户去过后，可以很轻地把验证发布出来；**
4. **发现新地点时，可以直接发布而不被审核拖延；**
5. **看到不准信息时，可以一键反馈，让地点状态自适应更新。**
